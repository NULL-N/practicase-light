# reports/ — 報告書・振り返りの置き場

課題で書く報告書はすべてここに置く(完了報告・振り返りなど)。

- ファイル名の例: `T-000_setup_report.md` / `T-001_fix_report.md` / `T-001_retrospective.md`
- テンプレートは `docs/templates/` にある
- このフォルダはどの課題でも変更してよい(チェックツールの scope 検査で常に許可されている)

## 最初の対応表(迷ったらまずここ)

| 場面 | 成果物 | 使用するテンプレート |
| --- | --- | --- |
| T-000 完了報告 | `reports/T-000_setup_report.md` | `docs/templates/setup_report.md` |
| T-001 の PR 本文 | Pull Request 本文 | `docs/templates/fix_report.md` |
| T-001 の見積・振り返り | `reports/T-001_retrospective.md` | `docs/templates/retrospective.md` |

**T-001 は 2 種類の成果物**(PR 本文=fix_report、振り返り=retrospective)を使う点に注意する。
T-002 以降のテンプレート対応は各チケットの ticket / rubric を参照する。

報告書はあなたの学習の記録そのものになる。積み上がった reports/ が、そのままポートフォリオの材料になる。
