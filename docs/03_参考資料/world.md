# PractiCase の世界(世界観設定)

この教材に登場するサービス・企業・人物・出来事はすべて架空であり、実在のものとは関係ない。

---

## 1. サービス「PractiCase」

**PractiCase(プラクティケース)** は、クライアント企業がエンジニア向けの短期・単発案件を掲載し、
エンジニアが応募して働ける、ITスポット案件マッチングサービスである。

- クライアント企業: 案件(時間単価・募集人数・応募締切など)を掲載し、応募者を承認・却下する
- エンジニア: 案件を検索して応募する。承認されると稼働に進む
- 運営(NullWorks): サービス全体を管理する

将来的には稼働報告 → 検収 → 報酬確定(精算)までを扱う(現在開発中の領域)。

## 2. 運営会社「株式会社NullWorks」

PractiCase を開発・運営している架空の会社。**あなた(学習者)は NullWorks の開発チームに配属されたエンジニア**である。
チームはチケット駆動で開発しており、あなたには業務チケット(tickets/)が割り当てられる。

## 5. チームの約束(NullWorks 開発チーム)

1. 仕事はチケットで受け、チケットで返す(口頭依頼は受けない)
2. 着手前に仕様書(docs/01_設計資料/)と既存コードを読み、影響範囲を考える
3. 変更はチケットの範囲に収める。ついで修正をしない
4. レビューはコードに対して行い、人に対して行わない。指摘には根拠(規約 ID・仕様番号)を添える
5. 報告は「結論 → 根拠 → 次の行動」の順で書く

## 6. この世界でのあなたの仕事の流れ

```text
チケットを読む → 仕様書・既存コードを読む → 影響範囲を考える
→ ブランチを切って作業する → チェックを実行する → Pull Request で提出する
→ レビュー・debrief と突き合わせる → 修正する → 振り返りを書く
```

チケットの文面は、早瀬リーダー・小野寺さん(CS 経由の顧客報告)など、実務で受け取る形を模している。
「誰から来た依頼か」も読み取りの一部として扱うこと。
