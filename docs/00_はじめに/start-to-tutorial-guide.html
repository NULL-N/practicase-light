導入からチュートリアル2完了までの手順書

この文書は、PractiCase を手元に用意してから T-000、tutorial、tutorial-2 を終えるまでの一本道です。印刷するか、ブラウザの別ウィンドウで開いたまま進めてください。PDF保存する場合は Ctrl+P を使います。

完了の定義

項目完了状態
環境docker compose --profile redmine up -d で app・pcp・Redmine が起動している
DBdocker compose exec app php tools/init-db.php を実行済み
Redminebootstrap とこの版の全課題(Master 65件 / Light 33件)の seed を実行済み

tutorialcheck tutorial が PASS、3行報告が reports/ にあり、Redmine で Closed

0. 使う場所を決める

作業は、必ず自分のコピーで行います。配布元や master 原本で課題を解かないでください。

GitHubを使う場合は、次の順で自分用リポジトリを作ります。

  1. ブラウザで、この教材の配布元GitHubリポジトリを開く
  2. Use this template → Create a new repositoryを選ぶ
  3. Ownerに自分のアカウント、Repository nameに任意の名前を指定し、Create repositoryを押す
  4. 作成された自分用リポジトリをcloneする
git clone https://github.com/<自分のアカウント>/<リポジトリ名>.git
cd <リポジトリ名>

GitHub を使えない場合だけ、ZIP を展開してローカルリポジトリ化します。

git init -b main
git add .
git commit -m "T-000: セットアップ開始"

1. VS Code でリポジトリの一番上を開く

VS Code の「ファイル → フォルダーを開く」で、README.md と docker-compose.yml がある階層を開きます。下位フォルダだけを開くと、docs/ や packs/php/tickets/ が検索範囲外になります。ファイルを探すときは Ctrl+P → ファイル名 が基本です。

2. 任意でチューター拡張を入れる

チューター拡張は任意です。入れなくてもすべての課題は完走できます。

  1. VS Code で Ctrl+Shift+P
  2. Extensions: Install from VSIX を選ぶ
  3. extensions/practicase-tutor/ フォルダにある .vsix ファイルを選ぶ
  4. Ctrl+Shift+P → Reload Window

この拡張はローカル完結で、外部通信はしません。

3. アプリを起動する

docker compose --profile redmine up -d
docker compose exec app php tools/init-db.php

Redmine を初期化し、この版に含まれる全課題を投入します。

PowerShell:

Get-Content -Raw tools/redmine/bootstrap.rb | docker compose --profile redmine exec -T redmine sh -c 'SECRET_KEY_BASE="$REDMINE_SECRET_KEY_BASE" bin/rails runner -'
docker compose exec -T app php tools/redmine-seed.php --ticket-root=packs/php/tickets --all

Git Bash:

docker compose --profile redmine exec -T redmine sh -c 'SECRET_KEY_BASE="$REDMINE_SECRET_KEY_BASE" bin/rails runner -' < tools/redmine/bootstrap.rb
docker compose exec -T app php tools/redmine-seed.php --ticket-root=packs/php/tickets --all

ブラウザで次を開きます。

http://localhost:8180/login.php

docs/03_参考資料/world.md の利用者名簿にあるアカウントでログインできれば起動成功です。

kiryu@example.com / password123

別のタブで Redmine を開き、教材用アカウントでログインします。

http://127.0.0.1:8280
practicase / practicase123

4. check コマンドを確認する

docker compose exec app php tools/check.php

課題一覧が出れば、チェックツールは動いています。以後、文中の check tutorial は次の意味です。

docker compose exec app php tools/check.php tutorial

5. T-000 を完了する

開くファイル:

packs/php/tickets/00_はじめに/T-000_setup/ticket.md
  1. Redmine で PractiCase Ticket ID = T-000 のチケットを開く
  2. 自分を担当者にし、見積をコメントして送信し、New → In Progress にする
  3. VS Code へ戻り、Redmine の URL 末尾にあるチケット番号を使って branch を作る 
    git switch main
git status --short
git switch -c feature/redmine-<チケット番号>-T-000-setup
  4. 起動確認を行う
  5. docs/templates/setup_report.md を reports/T-000_setup_report.md へコピーして、完了報告を書く
  6. check を実行する
docker compose exec app php tools/check.php T-000

結果: PASS になったら、報告を commit し、main へローカル merge します。

git add reports/T-000_setup_report.md
git commit -m "T-000: セットアップ完了を報告"
git switch main
git merge --no-ff --no-edit feature/redmine-<チケット番号>-T-000-setup
git branch -d feature/redmine-<チケット番号>-T-000-setup
git status --short

最後に Redmine へ戻り、その結果と報告ファイル名をコメントして送信し、In Progress → Resolved → Closed にします。

  • Redmine で T-000、tutorial、tutorial-2 が Closed

困ったとき

症状見る場所
ファイルが見つからないVS Code でリポジトリの一番上を開いているか確認
ポートが使えないPRACTICASE_PORT を変える
DB を戻したいdocker compose exec app php tools/init-db.php
Redmine が起動しないdocs/02_作業ルール/redmine-guide.md の fallback と reset を確認
check が FAILFAIL のメッセージを読み、対象ファイルと条件を確認
チュートリアル拡張が無い拡張なしでも ticket.md の手順で進められる

fallback: Redmine が起動しない・接続できない場合だけ、各 ticket.md の front matter を open → in_progress → resolved → closed と更新して進めます。Redmine 復旧後の進捗は自動同期されないため、必要な内容を手動で合わせてください。
