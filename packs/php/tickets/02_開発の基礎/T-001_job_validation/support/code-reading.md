# T-001 コードリーディング(既存コードの読み方)

`ProjectValidator` の `validate()` を読むための補助資料です。
**バグの答えは書いていません。** 正しく書けている「手本」の部分を1行ずつ解説するので、
これを読み方の型にして、壊れている項目を自分で読み解いてください。
(コード全体の地図は `docs/03_参考資料/code-tour.md`。この資料はその一段細かい「1メソッドの精読」です)

## 前提: validate() は何をするメソッドか

案件登録フォームから来た入力(`$input`)を1項目ずつ検証し、次の2つを組み立てて返します。

- `$errors` … 項目名 => エラーメッセージ の配列。**1件でもあれば登録させない**
- `$values` … 検証を通した後の値。エラーが無ければ、これがそのまま登録に使われる

つまり validate() の仕事は「入力を見て、ダメなものに札(errors)を付け、OKな値を清書(values)する」ことです。

## 手本を読む: title(案件名)の検証

```php
$title = trim((string) ($input['title'] ?? ''));
if ($title === '') {
    $errors['title'] = self::MSG_TITLE_REQUIRED;
} elseif (mb_strlen($title) > 100) {
    $errors['title'] = self::MSG_TITLE_TOO_LONG;
}
$values['title'] = $title;
```

1行ずつ:

| コード | 何をしているか |
|---|---|
| `$input['title'] ?? ''` | フォームに title があればその値、無ければ空文字。`??` は「左が無ければ右」(null合体演算子) |
| `(string)` | 文字列として扱うと明示(想定外の型が来ても落ちないように) |
| `trim(...)` | 前後の空白を除去。「スペースだけの入力」を空とみなすため |
| `if ($title === '')` | 空かどうかを厳密比較(`===`)で判定。空なら「必須です」のエラー |
| `self::MSG_TITLE_REQUIRED` | エラー文言。このクラス上部に定数で定義されている(文言を1箇所で管理する。規約 FLOW-3) |
| `elseif (mb_strlen($title) > 100)` | 空でなければ、次に長さを見る。`mb_strlen` は日本語も1文字と数える文字数(バイト数ではない) |
| `$values['title'] = $title` | エラーの有無に関わらず、清書した値を保存(エラー時は入力保持のため。仕様 G-7) |

**読み取れる型**: 「①入力を取り出す → ②複数のルールを if / elseif で順に見る → ③清書して保存」。
title は「必須」と「100文字以内」の**2つのルール**を、この形で1つずつチェックしています。

## 同じ型で description(案件内容)も読める

description のブロックは、title とほぼ同じ構造です。違いは長さ上限(2000)とエラー文言だけ。
**title の読み方がそのまま通用します。** 自分で読んで、同じ「型」になっていることを確認してください。

## ここから自分で: hourly_rate(時間単価)を読む

  title が `mb_strlen` を使ったように、hourly_rate は何を使うべき? いま使っている?
