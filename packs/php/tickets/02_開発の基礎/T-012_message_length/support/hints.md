# T-012 段階ヒント(3段)

詰まってから開きます。1段読んだら閉じて、15分は自分で。

---

## Hint 1: 型はもう持っている

T-001 でやった順番そのままです: **仕様の数字を読む → 境界を決める → その検証がコードのどこにあるべきかを探す**。
F-05 の message の行には、数字だけでなく**超過時の文言**も書いてあります。文言は探し物の強力な手がかりです —
tutorial で覚えた技(画面や仕様の文字で全文検索)は、ここでも効きます。

---

## Hint 2: 場所が T-001 と違う

T-001 の検証は Validator クラスにいました。今回の応募処理は
`create.php → ApplicationService::apply()` と流れます(code-tour.md)。
apply() の中には、すでに検証がいくつか並んでいます(A-2〜A-4)。
**仕様にあるのに、そこに並んでいないもの**は何か — 仕様の表とコードを1行ずつ突き合わせてください。

---

## Hint 3: 手がかりは「残骸」

このクラスの定数の一覧を上から眺めてください。**定義されているのに、どこからも使われていない定数**が1つあります
(T-001 のときは「定義されているのに呼ばれていないメソッド」でした — 同じ匂いです)。
それが見つかれば、直し方は既存の検証と同じ型 + その定数、だけです。

これ以上のヒントはありません。
