# T-029 セルフレビュー(rubric)

check の PASS は「事実が正しく読めた」ことしか保証しません。
提出前に、自由文2項目を自分でレビューしてください。

## direct_cause(直接原因)

- [ ] 「Flash.php が壊れている」で止まっていない(`at` は例外の発生場所にすぎない)
- [ ] `message` の `called in ... application_decide.php on line 40` を根拠に、
      **null を渡した呼び出し側**に欠陥があると特定できている
- [ ] 「結果が null(該当する応募が見つからない)の場合の分岐が無い」という
      因果まで書けている(型エラーは結果であって原因ではない)

## remediation_plan(修正方針)

- [ ] 最小限の修正方針になっている(大改修を提案していない)
- [ ] 「自分ではまだ直さない」前提の書き方になっている
- [ ] 修正後の回帰確認に触れている(同じ操作が 404 になること・
      正常な承認/却下が壊れていないこと)

## 調査の姿勢

- [ ] 事実(ログに書いてあること)と解釈(自分の推測)を区別して書いた
- [ ] 3件のエラーが「同じ1人のリトライ」だとログから根拠を持って言える
      (request_id は別、path / user は同じ)
- [ ] 問い合わせ文の「別の応募は普通に承認できた」とログの正常な
      `application.decide` 行が矛盾しないことを確認した
