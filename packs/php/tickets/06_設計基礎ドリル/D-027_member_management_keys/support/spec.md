# D-027 spec: 主キー・外部キーメモの型

## 提出物の型(reports/D-027_keys_note.md)

この節立てをそのまま使ってください。**テーブル名はこれまでと同じに**してください。
それぞれの言葉(`PRIMARY KEY`・`PRIMARY KEYにしないもの`・`FOREIGN KEY`)をそのまま
使うのが確実です。区切りには全角ダッシュ「—」を使います。

```markdown
# D-027 主キー・外部キーメモ

## テーブルごとの主キー・外部キー

### テーブル: <部署のテーブル名>
主キー・外部キー:
- PRIMARY KEY: <カラム名> — <理由>

### テーブル: <メンバーのテーブル名>
主キー・外部キー:
- PRIMARY KEY: <カラム名> — <理由>
- PRIMARY KEYにしないもの: email — <理由>
- PRIMARY KEYにしないもの: name — <理由>
- FOREIGN KEY: <部署への参照カラム> → <部署テーブル名>.id — <理由>
- 削除時の挙動: <部署が削除されようとした時、所属メンバーがいたらどうするか>
```

## 考え方のヒント

- **主キー(PRIMARY KEY)**: 「この行を1つに特定するには、どのカラムを見ればよいか」。
  D-024で数値型にした`id`のような、行ごとに自動で振られる番号が最も安定します
- **主キーに向かない情報**: 「値として変わりうるもの」「重複しないが人間が
  管理する情報」は主キーに向きません。メールアドレスは変更されることがありますし、
  氏名はそもそも重複しえます
- **外部キー(FOREIGN KEY)**: 「このカラムの値は、別のテーブルのどの行を指しているか」。
  メンバーテーブルの部署参照カラムは、部署テーブルの主キー(`id`)を指します
- **削除時の挙動**: 参照されている行(この場合は部署)を消そうとした時、
  参照している行(その部署のメンバー)がまだ存在するなら、多くの場合は
  「削除させない」のが安全です
