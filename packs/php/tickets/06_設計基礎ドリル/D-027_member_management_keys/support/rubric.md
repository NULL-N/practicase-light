# D-027 セルフチェック(rubric)

- [ ] 部署テーブル・メンバーテーブルとも、`id`相当を主キーにしたか
- [ ] メールアドレスを主キーにせず、なぜかの理由を書いたか(変わりうる情報だから、等)
- [ ] 氏名を主キーにせず、なぜかの理由を書いたか(同姓同名がありうるから、等)
- [ ] メンバーテーブルから部署テーブルへの外部キーを設計したか
      (どのカラムが、どのテーブルの、どのカラムを指すか)
- [ ] 部署削除時の挙動(所属メンバーがいたらどうするか)に触れたか
- [ ] ER図やCREATE TABLE文の話が主題になっていないか(それは次段階以降の話)
- [ ] 給与・勤怠・評価・個人番号・住所・健康情報のいずれにも触れていないか
