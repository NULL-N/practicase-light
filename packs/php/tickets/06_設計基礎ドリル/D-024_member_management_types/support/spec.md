# D-024 spec: 型選択メモの型

## 提出物の型(reports/D-024_column_types_note.md)

この節立てをそのまま使ってください。**テーブル名はD-022・D-023で使ったものと同じに**
してください。各カラムは「カラム名: 型 — 理由」の形で、区切りには全角ダッシュ「—」を
使います。

```markdown
# D-024 型選択メモ

## テーブルごとの型選択

### テーブル: <D-023で決めたテーブル名>
カラム型:
- <カラム名>: <型> — <なぜその型か、一言>
- <カラム名>: <型> — <理由>

### テーブル: <D-023で決めたテーブル名>
カラム型:
- <カラム名>: <型> — <理由>
- <カラム名>: <型> — <理由>

(D-023で洗い出したカラムの数だけ繰り返す)
```

## 型の考え方

- **文字列型**(TEXT・VARCHAR等): 名前・メールアドレスのような、決まった形の無い言葉の情報
- **数値型**(INTEGER等): 個数・番号・他のテーブルの`id`を指し示す参照
- **日時型**(DATE・DATETIME等): 「いつ」を表す情報
- **決まった種類の値**(ENUM、または文字列型で代用): 「在籍中/退職済み」のように、
  取りうる値があらかじめ決まっている情報

## 決めなくてよいこと

- NULLを許すか(D-025の仕事)
- 重複を禁止するか・UNIQUE(D-026の仕事)
- PRIMARY KEY・FOREIGN KEY(D-027の仕事)
- コード(この章では一切書かない)

これらに触れてしまっても check が落ちることはありませんが、この段階では「型と、その理由」
だけに集中してください。
