# D-029 セルフチェック(rubric)

- [ ] `CREATE TABLE`文が2つ以上あり、実際にSQLiteで実行が通るか(構文エラーが無いか)
- [ ] 部署名・氏名・メール・状態・部署参照のカラムが揃っているか(D-023)
- [ ] 型がD-024の判断どおりになっているか
- [ ] 部署名・氏名・メール・状態・部署参照がNOT NULLになっているか(D-025)
- [ ] 状態にDEFAULT(在籍中相当)があるか(D-025)
- [ ] メールにUNIQUE、氏名にはUNIQUEを付けていないか(D-026)
- [ ] 両テーブルにPRIMARY KEY、部署参照にFOREIGN KEYがあるか(D-027)
- [ ] 氏名にPRIMARY KEYを付けていないか(D-027)
- [ ] FKの削除時挙動がRESTRICT相当か(D-027)
- [ ] D-028のER図と、このDDLの内容が一致しているか
- [ ] 設計レビュー欄で、D-022〜D-028の各段階を実際に見比べたか
- [ ] 「気をつけたこと」または「見直したこと」を具体的に書いたか
- [ ] 給与・勤怠・評価・個人番号・住所・健康情報のいずれにも触れていないか
