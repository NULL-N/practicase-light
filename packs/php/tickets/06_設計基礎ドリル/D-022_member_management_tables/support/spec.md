# D-022 spec: テーブル洗い出しメモの型

## 提出物の型(reports/D-022_tables_note.md)

この節立てをそのまま使ってください。テーブルの数だけ「### テーブル: 」のブロックを繰り返します
(最低2つ)。

```markdown
# D-022 テーブル洗い出しメモ

## テーブル一覧

### テーブル: <テーブル名(英語・複数形が読みやすいですが必須ではありません)>
役割: <このテーブルが何のためにあるか、一行で>

### テーブル: <テーブル名>
役割: <一行で>

(必要なだけ繰り返す。最低2つ)
```

## 考え方のヒント

「社内メンバー管理」というお題を、**登場する「モノ」や「グループ」**に分解してみてください。
例えば「メンバー」というモノ自体は1つのテーブルになりそうです。では、メンバーが所属する
「部署」はどうでしょうか。メンバーの中に部署名を直接書くのと、部署を別テーブルにして
つなげるのとでは、何が違うでしょうか(この「つなげ方」はD-027で本格的に扱います。
ここでは「テーブルを分けた方が良さそうだ」という感覚だけで十分です)。

## 決めなくてよいこと

- カラム(列)の詳細(次の D-023 の仕事)
- 型・NULL・UNIQUE・PK/FK(D-024〜D-027 の仕事)
- コード(この章では一切書かない)
