# D-025 spec: NULL/DEFAULTメモの型

## 提出物の型(reports/D-025_null_default_note.md)

この節立てをそのまま使ってください。**テーブル名はこれまでと同じに**してください。
各カラムは「カラム名: NOT NULL(またはNULL可)、必要ならDEFAULT — 理由」の形で、
区切りには全角ダッシュ「—」を使います。

```markdown
# D-025 NULL/DEFAULTメモ

## テーブルごとのNULL/DEFAULT判断

### テーブル: <これまでのテーブル名>
NULL/DEFAULT:
- <カラム名>: NOT NULL — <理由>
- <カラム名>: NOT NULL, DEFAULT <初期値> — <理由>

### テーブル: <これまでのテーブル名>
NULL/DEFAULT:
- <カラム名>: NOT NULL — <理由>
- <カラム名>: NOT NULL, DEFAULT <初期値> — <理由>
- <カラム名>: NULL可 — <理由(なぜ空でもよいか)>

(これまで洗い出したカラムの数だけ繰り返す)
```

## 判断の考え方

- **NOT NULL(必須)にする情報**: そのカラムが空だと、業務として成立しない・意味が
  通らない情報(部署名・氏名・メールアドレス・状態・所属部署への参照、など)
- **NULL可(任意)にする情報**: 今は空で、条件が起きた時に初めて埋まる情報
  (退職日・利用停止日のような「起きたら記録する」情報)
- **DEFAULT(初期値)を検討する情報**: 新規登録の時点で、多くの場合こう決まっているはずの値
  (状態なら「在籍中」、登録日時なら「今この瞬間」)

## 決めなくてよいこと

- UNIQUE・重複禁止(D-026の仕事)
- PRIMARY KEY・FOREIGN KEY(D-027の仕事)
- コード(この章では一切書かない)

これらに触れてしまっても check が落ちることはありませんが、この段階では「空でもいいか」
「初期値は何か」だけに集中してください。
