# D-013 rubric(提出前のセルフチェック)

- [ ] URL・メソッド・パラメータが表になっている(文章の中に埋もれていない)
- [ ] エラー文言が「〜してください」の仕様書調で確定している(「エラーを出す」で止まっていない)
- [ ] Service / Repository / public の割り当てが具体名で書かれている(ARC-1〜3 に沿う)
- [ ] 「最大3つ」の守り場所が明記されている(D-4 の方針と矛盾していない)
- [ ] 絞り込みの値(id か名前か)を理由つきで選んでいる
- [ ] テスト観点に境界のペア(3個/4個 など)が2組以上ある
- [ ] 読み手が D-011 / D-012 を開かなくても実装に入れる(必要な決定がこの1枚に揃っている)
