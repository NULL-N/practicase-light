# D-011 hints(1段ずつ開く。15分粘ってから)

## 段1: 白紙から書かない

`docs/01_設計資料/features.md` の **F-03(案件一覧・検索)** を開いて、節の構成を観察してください。
「概要 → 表示対象 → 条件 → 仕様」という並びと文体を、そのまま F-20 の型として借ります。
設計書は作文ではなく、**チームの既存の型に合わせて書く文書**です。

## 段2: 4つの問いに1行ずつ答えてから清書する

- 誰が付ける? → client(S-12・登録時・最大3つ)
- 誰が使う? → engineer(S-03・1つ選んで絞る。未選択なら絞らない)
- 見えるのはどこ? → S-03 の一覧と S-04 の詳細
- 何をやらない? → 編集での付け直し/タグ管理画面/自由入力

この4行が書ければ、F-20 の骨格は完成しています。

## 段3: 「書かない」ことに耐える

「タグが4つ選ばれたときのエラーメッセージは?」— 気になりますが、**ここでは書きません**。
基本設計に文言まで書き込むと、詳細設計との境界が消え、後工程の人が「どこまで決まっているのか」を
判別できなくなります。「最大3つ」という**ルール**だけを書き、**文言**は D-013 に残してください。
