# D-010 spec: 要望整理の素材と型

## 入力(これがすべて)

ticket.md に引用した田淵さんのメール原文。それ以外の情報はまだ無い —
**無いことに気づいて質問に変えるのが、この課題の中心作業**です。

## 既に決まっている前提(これは聞かなくてよい)

- 現行の案件検索(S-03)の条件は「キーワード+リモート可のみ」(docs/01_設計資料/features.md の F-03)
- 「リモート可」は `is_remote` として**すでに存在する**(F-02 / F-03)
- タグ・ジャンルに相当する仕組みは、画面にも DB にもまだ何も無い(docs/01_設計資料/ を見て確認してよい)

## 提出物の型(reports/D-010_requirement_note.md)

この節立てをそのまま使ってください。

```markdown
# D-010 要望整理ノート

## 1. 要望の要約(3行以内)

## 2. 確認事項(田淵さんに聞くこと)
- 質問: …
  - なぜ聞くか: …(答えによって何が変わるかを書く)

## 3. 決定事項(現時点の仮決め+根拠)
- 仮決め: …
  - 根拠: …

## 4. やらないこと候補(今回のスコープ外にしたいもの)
- …
```

## 整理に使う観点(全部使う必要はない)

- その言葉(ジャンル)は**誰が・いつ・どの画面で**使うものか
- 数の上限はあるか(1案件にいくつ? 一覧の選択肢はいくつ?)
- 既にある仕組み(`is_remote`)と重複しないか
- 「直せると嬉しい」は、**何を・誰が**直す話か
