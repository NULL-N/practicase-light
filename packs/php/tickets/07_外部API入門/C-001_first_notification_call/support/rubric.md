# C-001 セルフレビュー基準

check が PASS したら、提出前に自分で以下を確認する。

## 操作(やったこと)

- [ ] `app` コンテナの中から PCP を呼んだ(ホストから直接叩こうとして詰まらなかったか)
- [ ] 401 / 400 のエラーを**わざと**一度は出してみた(エラーの形を自分の目で見たか)
- [ ] `queued` の状態と、遷移後(`delivered` / `failed`)の状態を両方見た

## 理解(説明できること)

- [ ] 「通知を作成する」と「状態を確認する」が別のリクエストである理由を説明できる
  (作成した瞬間には配送が終わっていない — 非同期の考え方)
- [ ] 監査ログに API キーの末尾4桁しか残らない理由を、自分の言葉で説明できる
- [ ] `/v1/health` が監査ログに残らない理由を説明できる

## 報告書

- [ ] 3つの見出しを指定の文言のまま使った
- [ ] 実際に返った通知ID(`ntf_...`)を書いた(架空のIDを書いていない)
- [ ] 「事実」と「読み取れる意図」を区別して書いた

## 提出前の共通確認

- [ ] 課題のcheckがPASSし、課題固有の成果物と報告内容を確認した
- [ ] commit・pushする前に、変更内容がこのrubricを満たしているか確認した

## 提出後の共通確認

- [ ] Pull Requestをmergeした
- [ ] merge後に`support/debrief/`がある場合は自分の提出と突き合わせた
- [ ] 突き合わせで見つけた違いを踏まえて振り返りを書いた
- [ ] RedmineへPASS結果・Pull RequestのURL・振り返りをコメントし、チケットを`Closed`にした
