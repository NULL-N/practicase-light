# C-003 セルフレビュー基準

check が PASS したら、提出前に自分で以下を確認する。

## 操作(やったこと)

- [ ] **同じ閲覧用キー**で、参照の成功(200)と作成の拒否(403)を両方出した
- [ ] 403 のレスポンスで `INSUFFICIENT_SCOPE` を確認した
- [ ] 監査ログで、suffix `3333` の SUCCESS(参照)と INSUFFICIENT_SCOPE(作成)が
      並んでいるのを見た

## 理解(説明できること)

- [ ] 401 と 403 の違いを一言で言える
      (401=誰か分からない / 403=誰かは分かるが権限が足りない)
- [ ] 「403 が返ってきた時点で認証は通っている」と説明できる(認証→認可の評価順)
- [ ] 403 のときの正しい対処(権限の付与を依頼する・権限のあるキーに切り替える)と、
      意味のない対処(リトライ・キーの綴り確認)を区別できる
- [ ] 閲覧用キーに作成権限が無いのは**発行ミスではなく正しい設計**だと説明できる
      (監視ツールに書き込み権限を渡さない = 必要最小限の権限)

## 報告書

- [ ] 3つの見出しを指定の文言のまま使った
- [ ] エラーコード(INSUFFICIENT_SCOPE)を書き写した
- [ ] 401 との対比が、原因と対処の両方で書かれている
