# C-002 セルフレビュー基準

check が PASS したら、提出前に自分で以下を確認する。

## 操作(やったこと)

- [ ] 成功(201)と失敗(401)の**両方**を自分のリクエストで出した
- [ ] 失敗を2種類(キー無し・誤ったキー)試した
- [ ] audit-log を `event_type` で絞り込んで確認した

## 理解(説明できること)

- [ ] 401 が「誰か分からない」状態であることを、自分の言葉で説明できる
- [ ] 401 は同じリクエストを再送しても直らない(直すのはキー・ヘッダーの側)と説明できる
- [ ] `api_key_suffix` が null になる場合と、末尾4桁が残る場合の違いを説明できる
- [ ] 認証(401)と認可(403)が別物であることを、一言で言える(403 は後続課題で扱う)

## 報告書

- [ ] 3つの見出しを指定の文言のまま使った
- [ ] エラーコード(INVALID_API_KEY)を書き写した
- [ ] 成功と失敗の**対照実験**になっている(失敗だけの報告になっていない)
