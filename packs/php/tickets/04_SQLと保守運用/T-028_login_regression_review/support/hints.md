# T-028 hints

## Hint 1: タグの意味

- `[bug]` — 直っていない・新しく壊れてしまった問題(コードの動きがおかしい点)
- `[impact]` — その `[bug]` が実務で起きたら何が困るか(誰に・どんな影響が出るか)
- `[fix]` / 対応方針 — どう直すべきか、具体的な直し方の方向性

## Hint 2: diff.patch を、行ごとに丁寧に読む

`-` で始まる行は削除された行、`+` で始まる行は追加された行です。**削除された行にも
情報があります**。「なぜこの行が消えたのか」「消えて困ることはないか」を考えてください。

`AuthService.php` の差分と、`T-019.php`(テストファイル)の差分、**2つのファイルが
変更されています**。両方見てください。

## Hint 3: 条件式を実際に手を動かして確認する

```php
if ($user['status'] !== 'active' && $user['role'] !== 'admin') {
    return null;
}
```

この1行に、`status`(active/suspended)と `role`(admin かどうか)の**2つの組み合わせ**
(2×2 = 4パターン)を、実際に紙に書くか頭の中で当てはめてみてください。
「suspended の admin」のケースが、どうなるか特に注目してください。
