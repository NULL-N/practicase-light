# T-000 トラブルシューティング(詰まったら見る)

この課題のヒントは、他のチケットと違って**答えを隠しません**。
環境構築の詰まりは我慢して学ぶものではないからです。ただし1つだけ練習してほしいことがあります —
**エラー文を読むこと**。赤い文字の塊から「Error / cannot / not found の直後」を探して読む。
それが、この先ずっと使う技能の最初の1つです(解決した詰まりは、完了報告のネタになります)。

---

## 「docker: command not found」「cannot connect」「pipe/socket が見つからない」

Docker Desktop が**起動していません**(インストールされていても、起動していなければ使えません)。
Docker Desktop を立ち上げ、クジラのアイコンが安定するまで待ってから、同じコマンドをやり直してください。
インストール直後なら、一度 OS の再起動が必要な場合があります。

## 「port is already allocated」「address already in use」

このパソコンの別のソフトが**同じポート(8180)を先に使っています**。
`docker-compose.yml` の冒頭コメントにあるとおり、環境変数 `PRACTICASE_PORT` で番号を変えられます
(例: 8280)。変えた場合、ブラウザで開く URL のポートも同じ番号になります。

## 起動したはずなのにブラウザに何も出ない

- URL は `http://localhost:8180`(ポートを変えた人はその番号)
- `docker compose ps` で `app` の STATUS が **Up** になっているか確認。
  ps は process status の略 — 「いま動いているか」の一覧です
- Up なのに出ないときは、`docker compose logs app` でエラーが出ていないかを見る(logs = 記録の表示)

## 「could not find driver」「no such table」など、画面がエラー文を出す

DB の初期化(`docker compose exec app php tools/init-db.php`)を実行しましたか?
setup-guide の手順は**順番に意味があります** — 起動(箱を用意)→ 初期化(中にデータを入れる)の順です。

## `check.php` や `init-db.php` が「そんなファイルは無い」と言われる

それらは**コンテナの中で**実行するスクリプトです。頭に `docker compose exec app` が付いているか
確認してください。exec は execute(実行)の略で、「動いているコンテナの中でこの命令を実行して」という意味 —
だからコンテナが起動している(`up -d` 済み)ことが前提です。
また、コマンドは**リポジトリのルート**(docker-compose.yml があるフォルダ)で実行します。

## PowerShell でコマンドが変な壊れ方をする

- 複数行に見えるコマンドをコピペすると途中で切れることがあります。**1行にしてから**貼り付けてください
- 作業フォルダのパスに**空白や日本語**が入っていると、ツールによっては誤動作します。
  `C:\dev\practicase` のような浅くて英数字だけの場所に clone するのが安全です

## どうにもならなくなったら

環境は使い捨てられます。setup-guide の「環境の後始末・作り直し」の手順で一度きれいに消して、
最初からやり直すのが結局いちばん早いことも多いです(それも実務でよくある判断です)。

解決したら、**何に詰まり、エラー文のどこを読み、どう解決したか**を完了報告に1行ずつ書いてください。
それは失敗の記録ではなく、あなたの環境の「取扱説明書」になります。
